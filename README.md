# APRS Net - Android

Android client for the [Advanced APRS Go Server](https://github.com/2E0LXY/Advanced-APRS-Go-server),
the companion to the [desktop APRS Client](https://github.com/2E0LXY/APRS-Client).

Built with [Capacitor](https://capacitorjs.com/) - wraps the full aprsnet.uk web
dashboard in a native Android shell with GPS, notifications and saved settings.

## Features

- Full APRS Net dashboard - live map, stations, messages, weather, leaderboard,
  ISS tracking, admin panel - the same site as the desktop client
- **Native GPS** - plots your own location using Android location services
- **Push notifications** - native Android notifications for APRS messages
- **Saved settings** - callsign, passcode, member account and appearance
  preferences persist between launches (Capacitor Preferences)
- **Single sign-on** - optional automatic login to your aprsnet.uk member account
- **Auto-connect** - skips the launch screen once set up
- Hardware back-button navigation
- Dedicated to www.aprsnet.uk - nothing to configure beyond your callsign

## Server

Like the desktop client, this app is dedicated to **www.aprsnet.uk**. The server
URL is fixed.

## Download

See [Releases](https://github.com/2E0LXY/APRS-Android/releases) for the signed
APK. Every push to `main` builds a debug APK via GitHub Actions; pushing a `v*`
tag publishes a release APK.

To install: download the APK, allow "install from unknown sources" for your
browser/file manager, and open it.

## Architecture

```
www/
  index.html        - Launch / settings screen
  bridge.js         - Native bridge: settings persistence, GPS, notifications;
                      exposes window.aprsClient (same API as the desktop preload)
capacitor.config.json - Capacitor project config (appId uk.aprsnet.client)
android-src/
  MainActivity.java - Adds the APRSNetApp User-Agent + enables DOM storage/GPS
.github/workflows/
  build.yml         - CI: builds the APK on every push
```

The app launches `www/index.html` (callsign + settings), then navigates the
WebView to the live `https://www.aprsnet.uk` dashboard. `bridge.js` installs
`window.aprsClient` - the same interface the desktop client's preload exposes -
so the site can request GPS fixes, raise notifications and read/write settings
through native Android APIs.

## Building from source

### Prerequisites
- Node.js 18+
- JDK 17
- Android SDK (API 34)

### Build

```bash
git clone https://github.com/2E0LXY/APRS-Android
cd APRS-Android
npm install
npx cap add android
npx cap sync android
cd android
./gradlew assembleDebug
# APK -> android/app/build/outputs/apk/debug/app-debug.apk
```

The `android/` folder is generated by `npx cap add android` and is not committed;
CI regenerates it on every build.
